# Sprint Planning Summary - Initial Foundation

This document summarizes the tickets created for the first approved sprint, focusing on establishing the foundational custom objects and their initial fields as per the Solution Design Document (SDD). These tickets represent Layer 1 (Foundation) of our development plan.

## Tickets Created:

### Ticket: CONOS-4
**Summary:** Create Student__c Custom Object and Fields
**Description:** This ticket covers the creation of the `Student__c` custom object along with its basic fields such as `Student_ID__c` (Auto Number), `Date_of_Birth__c`, `Email__c`, `Phone__c`, `Address__c`, `Enrollment_Date__c`, `Major__c` (Picklist), and `Status__c` (Picklist).

### Ticket: CONOS-5
**Summary:** Create Course__c Custom Object and Fields
**Description:** This ticket covers the creation of the `Course__c` custom object and its basic fields including `Course_Code__c` (Auto Number), `Description__c`, `Credits__c` (Number), `Max_Students__c` (Number), `Start_Date__c`, `End_Date__c`, and `Status__c` (Picklist).

### Ticket: CONOS-6
**Summary:** Create Instructor__c Custom Object and Fields
**Description:** This ticket covers the creation of the `Instructor__c` custom object and its basic fields such as `Instructor_ID__c` (Auto Number), `Email__c`, `Phone__c`, `Hire_Date__c`, `Rank__c` (Picklist), `Status__c` (Picklist), and `Office_Location__c`.

### Ticket: CONOS-7
**Summary:** Create Department__c Custom Object and Fields
**Description:** This ticket covers the creation of the `Department__c` custom object and its basic fields including `Department_Code__c` (Auto Number), `Description__c`, `Phone__c`, `Email__c`, and `Status__c` (Picklist).

### Ticket: CONOS-8
**Summary:** Create Building__c Custom Object and Fields
**Description:** This ticket covers the creation of the `Building__c` custom object and its basic fields such as `Building_Code__c` (Auto Number), `Address__c`, `Number_of_Floors__c` (Number), and `Capacity__c` (Number).

## Overall Progress:
These 5 tickets lay the critical foundation for the University Management System by establishing the core custom objects. The next steps will involve building out relationships and more complex data model elements.
